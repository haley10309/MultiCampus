1. 데이터 베이스 시스템 구성 요소
    1-1. 사용자 
        User/End User : 응용 프로그램을 사용하는 사람
        Application Programmer : DBMS와 연동된 응용 프로그램 개발하는 사람
        DBA : DBMS 관련된 하드웨어 / 소프트웨어 관리자

    1-2. 데이터 언어
        DDL(Data Definition Language) : 데이터 정의어 (create , alter, drop)
        DML(Data Manipulation Language) : 데이터 조작어 (insert, update , delete) -> 데이터 삽입, 삭제
        DCL(Data Control Language) : 데이터 제시어 (grant , revoke) -> 정확성 유지

    1-3. 3단계 스키마
        외부 스키마 : 개별 사용자 입장에서 데이터 베이스 구조를 기술한 것
        개념 스키마 : 조직 전체 입장에서 통합한 데이터베이스를 기술한 것
        내부 스키마 : 저장장치 입장에서 전체 데이터 베이스가 저장되는 방법을 명세

2. 데이터 모델 
    2-1. 구성요소
        데이터 구조 : 데이터 베이스에 저장할 데이터에 대해 요소와 관계를 명세한 것 (정적)
        연산 : 데이터 구조에 따라 실제 값들을 처리하느 작업에 대한 명세
        제약조건 : 데이터를 조작하는 한계 규정
    
    2-2.개체 종류
        유형 엔티티 : 물리적으로 실체, 지속적으로 사용 (책, 주문)
        개념 엔티티 : 물리적으로 존재 x (주문)
        사건 엔티티 : 비즈니스 프로세스 실행 시 생성 ( 배송정보, 결제 정보)

    2-3. 개체 발생 시점에 따른 종류 
        기본 엔티티 : 업무에 존재하는 정보 , 독립적으로 생성 (고객, 직원)
        중심 엔티티 : 기본엔티티에서 발생, 데이터 양 많음 , 다른 엔티티와의 관계를 통해 사건 엔티티 생성
        사건 엔티티 : 두개 이상의 엔티티로부터 발생, 내용 빈번히 변경 (HR 평가, 출퇴근 정보, 주문 , 구매, 배송)
    2-4. 키의 종류 (고유 식별자)
        수퍼키(super key) : 행을 고유하게 실별할 수 있는 속성
        후보키(Candidate key) : 행을 유일하게 실별할 수 있는 최소한의 속성들의 집합
        기본키(primary key) : 여러 후보키 중에서 하나를 선택해서 기준으로 사용하는 키
        대체키(Alternate key) : 후보키가 2개 이상, 그중하나를 기본키, 남은 후보키가 대체키
        외래키(Foreign key) : 다른 릴레이션의 기본키를 참조하는 속성 
    2-5. 무결성 제약 조건
        개체 무결성 제약조건 : 기본키는 null 이나 중복값을 가질 수 없음.
        참조 무결성 제약조건 : 자식 릴레이션 외래키는 부모 릴레이션 기본키와 동일해야 함. 
    2-6. ERD (Entity Relation Diagram)
        테이블과 뷰간의 관련성을 표시한 그림

3. 정규화
    3-1. 함수적 종속성 
        속성들간의 종속성 X-> Y
        X : 결정자 (학번)
        Y : 종속자 (학생이름, 학부)

    3-2. 단계
        3-2-1. 제1정규화 : 모든 속성의 도메일이 원자값이어야 함.
        3-2-2. 제2정규화 : 기본키와 기본키가 아닌 열에 종속관계가 있으면 별도의 테이블로 분리
        3-2-3. 제3정규화 : 기본키가 아닌 열들간에 종속관계가 있으면 별도의 테이블로 분리
